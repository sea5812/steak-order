# Business Rules - Unit 4: Frontend

## 1. 인증 규칙

### BR-AUTH-01: 테이블 자동 로그인
- localStorage에 `tableToken`이 존재하면 자동 인증 시도
- JWT 만료 시 저장된 credentials로 재로그인 시도
- 재로그인 실패 시 `/setup` 페이지로 리다이렉트
- 인증 정보가 없으면 `/setup` 페이지로 리다이렉트

### BR-AUTH-02: 관리자 세션 관리
- JWT 토큰은 localStorage에 저장
- 모든 관리자 API 요청에 `Authorization: Bearer <token>` 헤더 첨부
- 401 응답 수신 시 토큰 제거 + `/admin/login` 리다이렉트
- 16시간 세션 만료 시 자동 로그아웃

### BR-AUTH-03: 라우트 보호
- 고객 라우트 (`/`, `/order-list`, `/orders`): 테이블 인증 필요
- 관리자 라우트 (`/admin/dashboard`, `/admin/menus`, `/admin/tables`, `/admin/accounts`): 관리자 인증 필요
- `/setup`, `/admin/login`: 인증 불필요

## 2. 주문 목록 (장바구니) 규칙

### BR-CART-01: 항목 추가
- 동일 메뉴 추가 시 수량만 증가 (중복 항목 생성 안 함)
- 새 메뉴 추가 시 기본 수량은 모달에서 선택한 값 (기본 1)

### BR-CART-02: 수량 제한
- 최소 수량: 1
- 최대 수량: 99
- 수량 0 시도 시 항목 삭제 확인

### BR-CART-03: 금액 계산
- 항목 소계 = 단가 × 수량
- 총 금액 = 모든 항목 소계의 합
- 금액 표시 형식: "₩XX,XXX" (천 단위 콤마)

### BR-CART-04: 데이터 영속성
- 모든 변경 즉시 localStorage에 동기화
- 페이지 새로고침 시 localStorage에서 복원
- 주문 성공 시 localStorage 비우기

---

## 3. 주문 생성 규칙

### BR-ORDER-01: 주문 전 검증
- 주문 목록이 비어있으면 주문 버튼 비활성화
- 주문 확정 전 ConfirmDialog로 최종 확인

### BR-ORDER-02: 주문 성공 처리
- 성공 메시지: "주문이 접수되었습니다. 잠시 후 서빙해 드리겠습니다."
- 주문 번호 표시
- "확인" 버튼 클릭 시 메뉴 페이지로 이동 (자동 리다이렉트 없음)
- 주문 목록 자동 비우기

### BR-ORDER-03: 주문 실패 처리
- 에러 메시지 Toast로 표시
- 주문 목록 데이터 유지 (손실 방지)
- 재시도 가능

## 4. 관리자 대시보드 규칙

### BR-DASH-01: 뷰 모드
- 그리드 뷰 (기본): 테이블별 카드 형태
- 리스트 뷰: 테이블별 행 형태
- 뷰 모드 전환 시 데이터 유지

### BR-DASH-02: 실시간 업데이트
- SSE 연결 유지 (자동 재연결)
- 신규 주문: 해당 테이블 카드 하이라이트 (2초간 골드 보더 애니메이션)
- 주문 상태 변경: 즉시 반영
- 주문 삭제: 즉시 반영 + 총액 재계산
- 테이블 이용 완료: 카드 리셋

### BR-DASH-03: 주문 상태 변경
- 상태 전이: 대기중(pending) → 준비중(preparing) → 완료(completed)
- 역방향 전이 불가 (UI에서 제한)
- 상태 변경 시 `PUT /api/stores/:storeId/orders/:id/status` 호출

### BR-DASH-04: 주문 삭제
- 삭제 전 ConfirmDialog 필수
- 삭제 후 테이블 총 주문액 재계산
- 성공/실패 Toast 피드백

---

## 5. 메뉴 관리 규칙

### BR-MENU-01: 필수 필드 검증
- 메뉴명: 필수, 빈 문자열 불가
- 가격: 필수, 0 이상의 숫자
- 카테고리: 필수, 기존 카테고리 중 선택

### BR-MENU-02: 이미지 업로드
- 이미지는 선택사항
- 지원 형식: JPEG, PNG, WebP
- multipart/form-data로 전송

### BR-MENU-03: 순서 변경
- 위/아래 화살표 버튼으로 순서 조정
- 변경 후 서버에 새 순서 전송

---

## 6. 테이블 관리 규칙

### BR-TABLE-01: 이용 완료
- ConfirmDialog 필수: "테이블 X번 이용 완료 처리하시겠습니까?"
- 완료 시 해당 테이블의 현재 주문 목록 리셋
- 총 주문액 0으로 리셋
- 성공/실패 Toast 피드백

### BR-TABLE-02: 과거 내역
- 시간 역순 정렬
- 날짜 필터링 지원
- 각 주문: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각

---

## 7. UI/UX 규칙

### BR-UX-01: 터치 친화성
- 모든 인터랙티브 요소 최소 44×44px
- 버튼 간 충분한 간격 (최소 8px)

### BR-UX-02: 피드백
- 모든 API 호출 시 로딩 상태 표시
- 성공/실패 시 Toast 알림
- 위험한 작업 (삭제, 이용 완료) 시 ConfirmDialog

### BR-UX-03: 에러 처리
- 네트워크 에러: "서버에 연결할 수 없습니다. 잠시 후 다시 시도해주세요."
- 인증 에러 (401): 자동 로그아웃 + 로그인 페이지 리다이렉트
- 서버 에러 (500): "일시적인 오류가 발생했습니다."
- 검증 에러 (400): 서버 응답 메시지 표시

### BR-UX-04: 금액 표시 형식
- 한국 원화: "₩XX,XXX"
- 천 단위 콤마 구분
- 소수점 없음
