# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis Summary

| 항목 | 내용 |
|---|---|
| **User Request** | 디지털 테이블오더 서비스 구축 (고객 주문 + 관리자 운영) |

| **Scope Estimate** | System-wide (Frontend + Backend + Database) |
| **Complexity Estimate** | Complex (다중 사용자 유형, 실시간 통신, 세션 관리) |

---

## 1. 기술 스택

| 구성 요소 | 기술 | 선정 이유 |
|---|---|---|
| **Backend** | Node.js + Express (TypeScript) | 보일러플레이트 최소, SSE 구현 간편, 프론트와 언어 통일 |
| **Frontend** | React (TypeScript) + Vite | 빠른 개발 환경, HMR 지원 |
| **Database** | SQLite (better-sqlite3) | 설치 불필요, 파일 기반, 온프레미스에 적합 |
| **ORM** | Drizzle ORM | 경량, TypeScript 네이티브, SQLite 지원 우수 |
| **실시간 통신** | Server-Sent Events (SSE) | 단방향 실시간 통신에 적합, 구현 간단 |
| **인증** | JWT 토큰 기반 | 세션 관리 간편 |
| **배포 환경** | 로컬 서버 / 온프레미스 | 요구사항 그대로 유지 |
| **이미지 저장** | 로컬 파일 시스템 (서버 업로드) | 외부 의존성 없음 |
| **패키지 관리** | npm workspaces (모노레포) | 프론트/백엔드 통합 관리 |

```

         |                        |

            +-------+-------+
```

### 매장 관리 구조
- **초기**: 단일 매장 전용
- **향후**: 다중 매장(멀티테넌트) 확장 가능한 구조로 설계
- Store ID를 모든 엔티티에 포함하여 확장성 확보

### 동시 접속 규모
- **중규모**: 테이블 10~50개, 관리자 2~5명

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- **초기 설정** (관리자 1회 수행): 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- **자동 로그인**: 1회 로그인 성공 후 저장된 정보로 자동 로그인
- **세션 만료**: 16시간 후 자동 만료 (관리자와 동일)
- **로컬 저장**: 로그인 정보를 브라우저 로컬 스토리지에 저장

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- **프리미엄 UI 디자인 — "Black Marble" 컨셉**:
  - 다크 테마 기반 (제트블랙 #0A0A0A 배경, 다크그레이 #1A1A1A 카드)
  - 대리석 텍스처 느낌의 미묘한 그라데이션 포인트
  - 골드 #C9A96E 포인트 컬러 (버튼, 가격, 강조 요소)
  - 화이트 #F5F5F5 텍스트, 그레이 #999 보조 텍스트
  - 세리프 계열 폰트 사용 (메뉴명, 헤딩) — Playfair Display 또는 유사
  - 메뉴명과 설명 중심의 미니멀 레이아웃 (이미지는 보조적 역할)
  - 터치 친화적 버튼 (최소 44x44px)
  - 여백을 충분히 활용한 우아한 시각적 계층 구조
  - 서비스명 "Black Marble Table"이 헤더에 표시

#### FR-C03: 주문 목록 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 주문 목록 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행
- **용어**: "장바구니" 대신 "주문 목록(Order List)" 사용

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- **주문 성공 플로우**: 정중한 확인 메시지 표시 ("주문이 접수되었습니다. 잠시 후 서빙해 드리겠습니다.") → 주문 목록 자동 비우기 → 고객이 "확인" 버튼을 눌러 메뉴 화면으로 복귀 (자동 리다이렉트 없음)
- 주문 실패 시 에러 메시지 표시 및 주문 목록 유지
- **주문 정보**: 매장 ID, 테이블 ID, 주문 메뉴 목록(메뉴명, 수량, 단가), 총 주문 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- **주문 상태 실시간 업데이트** (SSE 기반, MVP 포함)
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 페이지네이션 또는 무한 스크롤

### 3.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증
- 16시간 세션 유지, 브라우저 새로고침 시 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 관리자 계정 관리
- 관리자가 추가 관리자 계정 생성 가능
- 매장별 관리자 계정 관리

#### FR-A03: 실시간 주문 모니터링
- **SSE 기반 실시간 주문 목록 업데이트** (2초 이내 표시)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링 기능

#### FR-A04: 테이블 관리
- **테이블 초기 설정**: 테이블 번호 및 비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 특정 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- **테이블 세션 처리**:
  - 세션 시작: 첫 주문 시 자동 시작
  - 세션 종료(이용 완료): 확인 팝업 → 주문 내역 과거 이력 이동 → 현재 주문/총액 리셋
  - 새 고객이 이전 주문 없이 시작 가능
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A05: 메뉴 관리 (CRUD)
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 (로컬 파일 업로드)
- 메뉴 수정
- 메뉴 삭제
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 주문 알림: 2초 이내 표시
- 메뉴 조회 API 응답 시간: 500ms 이내
- 주문 생성 API 응답 시간: 1초 이내
- 동시 접속: 테이블 50개 + 관리자 5명 동시 처리

### NFR-02: 가용성
- 매장 영업 시간 동안 안정적 운영
- 서버 재시작 시 세션 유지 (JWT 기반)

### NFR-03: 확장성
- Store ID 기반 멀티테넌트 확장 가능 구조
- 모든 엔티티에 store_id 포함

### NFR-04: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- **"Black Marble" 다크 테마**:
  - 배경: 제트블랙 #0A0A0A, 카드: #1A1A1A, 보더: #2A2A2A
  - 포인트: 골드 #C9A96E (버튼, 가격, 강조)
  - 텍스트: 화이트 #F5F5F5 (주), 그레이 #999 (보조)
  - 대리석 텍스처 느낌의 미묘한 그라데이션
- **타이포그래피**: Playfair Display 또는 유사 세리프 (메뉴명, 헤딩), Inter 또는 유사 산세리프 (본문, UI)
- 메뉴명·설명 중심의 미니멀 레이아웃, 이미지는 보조적 역할
- 충분한 여백과 우아한 시각적 계층 구조
- 반응형 디자인 (태블릿 최적화)
- 정중한 톤의 UI 문구 (고급 레스토랑에 어울리는 표현)
- 서비스명 "Black Marble Table" 헤더 표시

### NFR-05: 데이터 무결성
- 주문 생성 시 트랜잭션 처리
- 장바구니 데이터 클라이언트 측 로컬 저장
- 주문 이력 영구 보존

## 5. 제외 기능 (Out of Scope)

requirements/constraints.md에 정의된 대로 다음 기능은 구현하지 않음:
- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)

## 7. 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보 (store_id, name, ...)
- **Admin**: 관리자 계정 (admin_id, store_id, username, password_hash, ...)
- **TableInfo**: 테이블 정보 (table_id, store_id, table_number, password_hash, ...)
- **Category**: 메뉴 카테고리 (category_id, store_id, name, display_order, ...)
- **MenuItem**: 메뉴 항목 (menu_item_id, store_id, category_id, name, price, description, image_url, display_order, ...)
- **TableSession**: 테이블 세션 (session_id, table_id, store_id, started_at, ended_at, ...)
- **Order**: 주문 (order_id, store_id, table_id, session_id, total_amount, status, ordered_at, ...)
- **OrderItem**: 주문 항목 (order_item_id, order_id, menu_item_id, menu_name, quantity, unit_price, ...)
- **OrderHistory**: 과거 주문 이력 (이용 완료 시 이동)

### 용어 정의
- **테이블 세션**: 고객이 첫 주문 시작 ~ 이용 완료 처리까지의 기간
- **이용 완료**: 관리자가 테이블 세션을 종료하는 행위, 주문 내역이 과거 이력으로 이동됨
