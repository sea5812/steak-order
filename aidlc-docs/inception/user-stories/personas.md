# 테이블오더 서비스 - User Personas

---

## Persona 1: 고객 (Customer) — 익명 매장 방문자

| 항목 | 내용 |
|---|---|
| **역할** | 매장 방문 고객 (익명, 로그인 불필요) |
| **사용 환경** | 매장 테이블에 설치된 태블릿 (자동 인증 완료 상태) |

### 핵심 니즈
- 테이블에 앉으면 바로 메뉴를 보고 주문하고 싶다
- 주문 상태를 확인하고 추가 주문을 편리하게 하고 싶다

### 사용 시나리오
1. 테이블에 앉으면 태블릿에 메뉴 화면이 이미 표시되어 있음
2. 카테고리별로 메뉴를 탐색하고 장바구니에 추가
3. 장바구니에서 수량 조절 후 주문 확정
4. 주문 내역에서 상태를 실시간 확인
5. 필요 시 추가 주문 반복

## Persona 2: 매장 관리자 (Admin) — "박서연"

| 항목 | 내용 |
|---|---|
| **이름** | 박서연 |
| **나이** | 40대 |
| **역할** | 매장 운영 관리자 / 점주 |
| **기술 수준** | 기본적인 PC/태블릿 사용 가능 |
| **사용 환경** | 매장 카운터 PC 또는 태블릿 |

### 목표
- 들어오는 주문을 실시간으로 확인하고 처리하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 등록/수정/삭제하고 싶다
- 테이블 회전율을 효율적으로 관리하고 싶다

### 동기
- 인건비 절감 (주문 접수 인력 감소)
- 주문 누락/실수 방지
- 매장 운영 효율성 극대화
- 고객 만족도 향상

### 불편사항 (Pain Points)
- 바쁜 시간대에 주문을 놓치거나 실수한다
- 테이블별 주문 현황 파악이 어렵다
- 메뉴 변경 시 메뉴판을 다시 인쇄해야 한다
- 이전 고객의 주문과 현재 고객의 주문이 섞인다

### 사용 시나리오
1. 매장 오픈 시 관리자 로그인
2. 대시보드에서 테이블별 주문 현황 실시간 모니터링
3. 신규 주문 확인 후 상태 변경 (대기중 → 준비중 → 완료)
4. 고객 퇴장 시 테이블 이용 완료 처리
5. 필요 시 메뉴 등록/수정/삭제
6. 추가 관리자 계정 생성 (필요 시)
